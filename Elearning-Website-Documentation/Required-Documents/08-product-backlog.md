# E-Learning Website - Product Backlog

## 1. Purpose

This document lists the main features needed for the MVP and their priority.

## 2. Priority Levels

| Priority | Meaning |
| --- | --- |
| Must Have | Required for MVP |
| Should Have | Important, but can be delayed if needed |
| Later | Not part of MVP |

## 3. MVP Backlog

| ID | Feature | Role | Priority | Description |
| --- | --- | --- | --- | --- |
| PB-01 | Student registration/login | Student | Must Have | Student can create account and log in |
| PB-02 | Parent registration/login | Parent | Must Have | Parent can create account and log in |
| PB-03 | Parent-student linking | Parent | Must Have | Parent links to student using student ID |
| PB-04 | Teacher account approval | Admin | Must Have | Admin approves teacher accounts before course publishing |
| PB-05 | Education center account | Center/Admin | Must Have | Education center can manage its profile and teachers |
| PB-06 | Curriculum management | Admin | Must Have | Admin manages years, subjects, terms, and chapters |
| PB-07 | Course creation | Teacher | Must Have | Teacher creates course with year, subject, term, chapter, price, and description |
| PB-08 | Lesson management | Teacher | Must Have | Teacher adds video lessons to courses |
| PB-09 | MCQ quiz management | Teacher | Must Have | Teacher adds MCQ quizzes to courses |
| PB-10 | Course approval | Admin | Must Have | Admin approves or rejects courses |
| PB-11 | Course catalog | Student | Must Have | Student browses published courses |
| PB-12 | Course filtering | Student | Must Have | Student filters by year, subject, term, chapter, and teacher |
| PB-13 | Prepaid code generation | Admin | Must Have | Admin generates codes with value and serial number |
| PB-14 | Prepaid code redemption | Student/Parent | Must Have | Code adds value to one student's balance |
| PB-15 | Student balance management | System/Admin | Must Have | System tracks balance; admin can adjust it |
| PB-16 | Course purchase | Student | Must Have | Student uses balance to buy teacher course |
| PB-17 | Video lesson playback | Student | Must Have | Student watches enrolled course lessons |
| PB-18 | No-download video control | System | Must Have | Video should not be directly downloadable |
| PB-19 | Lesson completion tracking | Student/System | Must Have | Completion is manual or automatic after watching 90% |
| PB-20 | MCQ quiz solving | Student | Must Have | Student solves MCQ quizzes |
| PB-21 | Quiz score display | Student | Must Have | Student sees quiz score, no retry in MVP |
| PB-22 | Progress tracking | Student/Parent/Teacher | Must Have | Progress is visible to student, linked parent, and teacher |
| PB-23 | Admin code reports | Admin | Must Have | Admin views code generation and redemption reports |
| PB-24 | Enrollment reports | Admin/Teacher/Center | Must Have | Users view enrollment reports based on role |
| PB-25 | Teacher sales reports | Admin/Teacher/Center | Must Have | Sales reports are available based on role |
| PB-26 | Student progress reports | Admin/Teacher/Parent | Must Have | Progress reports are available based on role |
| PB-27 | Manual refund support | Admin | Should Have | Admin can reset student balance to 0 for approved refund |
| PB-28 | Admin price override | Admin | Should Have | Admin can edit teacher course prices |

## 4. Later Backlog

| ID | Feature | Reason |
| --- | --- | --- |
| L-01 | Mobile app | Not needed for first web MVP |
| L-02 | Live classes | Adds complexity |
| L-03 | Chat/community | Can be added after learning flow works |
| L-04 | Certificates | Not important for school-course MVP |
| L-05 | Subject bundles | Current scope is separate teacher course purchase |
| L-06 | Direct online payment gateway | MVP uses prepaid codes |
| L-07 | English interface | Arabic launches first |
| L-08 | Video watermarking | No-download is MVP protection |

## 5. Suggested Build Order

1. User accounts and roles
2. Curriculum setup
3. Teacher and course management
4. Course approval
5. Course catalog
6. Prepaid codes and student balance
7. Course purchase and enrollment
8. Video lessons and progress
9. MCQ quizzes
10. Parent view
11. Reports
