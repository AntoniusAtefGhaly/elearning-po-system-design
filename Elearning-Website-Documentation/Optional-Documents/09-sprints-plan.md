# E-Learning Website - Sprints Plan

## 1. Purpose

This document organizes the MVP backlog into simple Agile sprints.

## 2. Sprint Goal

Build a working Arabic e-learning MVP where students can buy teacher courses using prepaid balance, watch lessons, solve MCQ quizzes, and track progress.

## 3. MVP Sprint Scope

The MVP includes:

- Student, parent, teacher, education center, and admin accounts
- Curriculum setup
- Course creation and approval
- Course catalog
- Prepaid code redemption and student balance
- Course purchase and enrollment
- Video lessons
- Progress tracking
- MCQ quizzes
- Parent progress view
- Basic reports

## 4. Suggested Sprint Plan

| Sprint | Focus | Main Features |
| --- | --- | --- |
| Sprint 1 | Foundation | User accounts, roles, login, basic admin access |
| Sprint 2 | Curriculum and Teachers | Years, subjects, terms, chapters, teacher approval, education center setup |
| Sprint 3 | Courses | Course creation, lesson management, MCQ quiz setup, course approval |
| Sprint 4 | Catalog and Payment | Course catalog, filters, prepaid code generation, code redemption, student balance |
| Sprint 5 | Learning Flow | Course purchase, enrollment, video playback, lesson completion, progress tracking |
| Sprint 6 | Parent and Reports | Parent-student linking, parent progress view, code reports, enrollment reports, sales reports, progress reports |
| Sprint 7 | Testing and Release | Bug fixing, final testing, content review, MVP release preparation |

## 5. Main Dependencies

| Feature | Depends On |
| --- | --- |
| Course creation | Teacher account and curriculum setup |
| Course catalog | Approved courses |
| Course purchase | Prepaid code redemption and student balance |
| Lesson viewing | Course enrollment |
| Progress tracking | Lesson viewing and quiz solving |
| Parent progress view | Parent-student linking and student progress |
| Reports | Users, courses, codes, enrollments, and progress data |

## 6. MVP Exit Criteria

The MVP is ready when:

- Students can register and log in.
- Parents can link to students using student ID.
- Teachers can create courses and submit them for approval.
- Admin can approve teachers and courses.
- Admin can generate and manage prepaid codes.
- Student or parent can redeem a code to add student balance.
- Student can buy a course using balance.
- Student can watch lessons after enrollment.
- Lesson progress works manually and automatically after 90% video watching.
- Student can solve MCQ quizzes and view score.
- Parent can view linked student progress.
- Admin and teachers can view basic reports.

## 7. Later Sprints / Releases

Later sprints or releases may include:

- English interface
