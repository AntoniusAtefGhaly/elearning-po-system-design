# E-Learning Website - Data Model

## 1. Purpose

This document lists the main data objects needed for the MVP.

It is not a detailed database design. It helps the software team understand what data the system should store.

## 2. Main Entities

| Entity | Description |
| --- | --- |
| User | Basic login account for student, parent, teacher, center, or admin |
| Student | Student profile linked to a user account |
| Parent | Parent profile linked to a user account |
| Teacher | Teacher profile linked to a user account |
| Education Center | Center that can manage teachers and courses |
| Course | Teacher course that students can buy |
| Lesson | Video lesson inside a course |
| Quiz | MCQ quiz linked to course or lesson |
| Enrollment | Student access to a purchased course |
| Progress | Student lesson/course progress |
| Prepaid Code | Code generated by admin with value and serial number |
| Code Redemption | Record of code redeemed for a student |
| Student Balance | Current balance for a student |
| Balance Transaction | Record of balance changes |
| Report | Data used for codes, enrollments, sales, and progress reports |

## 3. Key Fields

| Entity | Important Fields |
| --- | --- |
| User | ID, name, phone, email, password, role, status |
| Student | ID, user ID, student ID, secondary year |
| Parent | ID, user ID |
| Teacher | ID, user ID, subject, bio, profile image, documents, approval status, education center ID |
| Education Center | ID, name, contact info, approval status |
| Course | ID, teacher ID, year, subject, term, chapter, title, description, price, approval status |
| Lesson | ID, course ID, title, video URL, order |
| Quiz | ID, course ID or lesson ID, question, answers, correct answer |
| Enrollment | ID, student ID, course ID, purchase amount, enrollment date |
| Progress | ID, student ID, course ID, lesson ID, completion status, quiz score |
| Prepaid Code | ID, code, serial number, value, status |
| Code Redemption | ID, code ID, student ID, redeemed by user ID, redeemed date |
| Student Balance | ID, student ID, current balance |
| Balance Transaction | ID, student ID, amount, type, reason, created by, date |

## 4. Main Relationships

| Relationship | Rule |
| --- | --- |
| User to Student | One user can have one student profile |
| User to Parent | One user can have one parent profile |
| User to Teacher | One user can have one teacher profile |
| Parent to Student | One parent can link to one or more students using student ID |
| Education Center to Teacher | One center can have many teachers |
| Teacher to Education Center | One teacher can be independent or linked to one center |
| Teacher to Course | One teacher can create many courses |
| Course to Lesson | One course can have many lessons |
| Course to Enrollment | One course can have many enrollments |
| Student to Enrollment | One student can enroll in many courses |
| Student to Balance | One student has one balance |
| Prepaid Code to Redemption | One code can be redeemed once for one student |
| Student to Progress | One student has progress records for lessons/quizzes |

## 5. Balance Rules

- Code redemption adds value to student balance.
- Balance can be used across multiple courses.
- Balance cannot become negative.
- Course purchase subtracts course price from balance.
- Admin can manually add or subtract balance.
- Refund resets balance to 0 only if the student has not used/bought a course.

## 6. Prepaid Code Rules

- Code has value and serial number.
- Code does not expire.
- Code can be active, used, or cancelled.
- Cancelled or used code cannot be redeemed.
- Code is manually distributed in MVP.
- One code can be redeemed for one student only.

## 7. Quiz and Progress Rules

- MVP quizzes are MCQ only.
- Quiz score is shown to student.
- Quiz retry is not allowed.
- Quiz result affects progress.
- Lesson completion can be manual.
- Lesson completion can be automatic after watching 90% of video.
