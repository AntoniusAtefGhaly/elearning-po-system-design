# E-Learning Website - Business Model

## 1. Purpose

This document explains the basic business idea for the e-learning website.

This is a Product Owner training project, not a real business launch.

## 2. Product Idea

An Arabic-first e-learning website for Egyptian secondary school students.

Students can browse teacher courses, redeem prepaid codes, enroll in courses, watch recorded lessons, solve simple quizzes, and track progress.

Teachers can publish courses based on the official Egyptian curriculum.

## 3. Target Users

| User | Description |
| --- | --- |
| Student | Learns online and enrolls in teacher courses |
| Parent | Supports payment and views student progress |
| Teacher | Creates and sells courses |
| Admin | Manages platform users, courses, codes, and approvals |

## 6. MVP Revenue Model

- Admin generates prepaid codes with fixed values, such as 50 EGP and 100 EGP.
- Student or parent redeems a code for one student only.
- Redeemed code value is added to the student's balance.
- Student uses balance to buy a teacher course.
- Balance can be used across multiple courses and cannot become negative.
- Codes are distributed manually for the MVP.
- Codes do not expire, but admin can cancel active codes.
- Codes have serial numbers for manual tracking.
- Refunds are handled manually: admin resets student balance to 0, then money is returned outside the system.
- Refund is allowed only before the student buys/uses a course.
- Platform can take commission from teacher course sales.

## 7. MVP Includes

- Student, parent, teacher, and admin accounts
- Course catalog by year, subject, term, chapter, and teacher
- Prepaid code redemption
- Course enrollment
- Recorded video lessons
- Basic assessments/quizzes

- Parent progress view
- Teacher course management
- Admin approval for teachers and courses
